\caption{A directed acyclic graph of build tasks. An arrow $a \to b$
reads ``$a$ must complete before $b$.'' A \engterm{topological sort}
lists the vertices so every edge points forward; here $a, b, c, d,
e, f$ is one valid order, and $a, c, b, d, e, f$ is another. A DAG
admits a topological order if and only if it has no directed cycle;
a cycle is a circular dependency that no linear schedule can honour.
The order is computed in $\Theta(n + m)$ by depth-first search with
finish-time ordering, or by repeatedly removing a source vertex
(Kahn's algorithm). Build systems, spreadsheet recalculation, and
course-prerequisite planning all run a topological sort.}

\caption{A set-cover instance on twelve ground elements. The optimum
covers all twelve with the three row-sets $S_1,S_2,S_3$ (red). A
greedy algorithm that always takes the set covering the most
still-uncovered elements may first grab the large diagonal set $S_A$
(blue, six elements), then need several small sets to mop up the
remainder, using more sets than the optimum. The greedy solution is
never worse than a factor $\ln n$ above optimal, and no
polynomial-time algorithm beats $(1-o(1))\ln n$ unless
$\mathrm{P}=\mathrm{NP}$; the bounded overshoot is what turns the
heuristic from a hope into a contract.}

# Data assets - Volume II, Chapter 1 (Algebra and proof discipline)

Small reference datasets used by the chapter's figures, code scripts,
and exercises. Each file is small enough to ship with the chapter
source; large outputs of the simulation scripts are written to this
directory at run time and are not committed.

## Files

| File | Source | Purpose |
|---|---|---|
| `pascal_to_6.csv` | Generated by `code/binomial_table.py` with `max_n=6`. | Reference data for the Pascal's triangle figure (figure 1.2 of this chapter); checked into the repository so that the figure can be cross-validated against the same CSV the simulation exercise produces. |
| `mco-unit-handoff.csv` | Authors' summary of the Mars Climate Orbiter Mishap Investigation Board report, current as of 2026. | Four-line audit-table format the chapter recommends for an interface specification. Demonstrates the conversion-factor column that, had it been required on the actual MCO interface, would have caught the unit slip. |
| `horner-vs-naive-ops.csv` | Counted by hand from the two evaluation algorithms in `code/horner_eval.py`. | Operation counts (multiplications, additions) for Horner's nested form versus the naive term-by-term sum, as a function of polynomial degree. Shows the linear-versus-quadratic growth in multiplications. |
| `welford-vs-naive.csv` | Generated by `code/welford_variance.py`. | Relative error of the running variance, Welford update versus the naive sum-of-squares single-pass identity, against a well-conditioned two-pass reference, as the data cluster offset grows from 0 to 1e8 at fixed spread 1e-3. Shows the naive identity's catastrophic cancellation (relative error growing past 1e6) while Welford stays near machine epsilon. |

## Provenance

`pascal_to_6.csv`: reproducible from `binomial_table.py 6 ../data/pascal_to_6.csv`.

`mco-unit-handoff.csv`: distilled from the Mars Climate Orbiter
Mishap Investigation Board Phase I Report (10 November 1999), entry
`acc:nasa-mco-mib` in `bibliography/references.bib`. The `quantity`
column lists representative variables crossing the contractor-to-
operator interface; the small-forces row is the variable whose
unaudited conversion caused the loss. The other rows are
representative of variables that did carry consistent units.

`welford-vs-naive.csv`: reproducible from `welford_variance.py`, which
fixes its random seed (`20260606`). Each row is one cluster offset; the
two-pass column is the well-conditioned reference, and the relative-
error columns quantify the cancellation gap the section 1.4 case study
predicts algebraically.

# Data assets — Volume VI, Chapter 11 (Bioinformatics)

## Files

| File | Source | Used by |
|---|---|---|
| `genome_a.fasta` | Synthetic small bacterial reference fragment, ~5 kb, modelled after a 16S rRNA + housekeeping-gene cluster (editor-generated educational data; not from a deposited record). | Project (align two bacterial genomes); calculation exercise on alignment statistics. |
| `genome_b.fasta` | Synthetic small bacterial query fragment, ~5 kb, with introduced SNPs and a short indel relative to `genome_a.fasta`. | Project (align two bacterial genomes); variant-calling exercises. |
| `example.vcf` | Synthetic VCF with 50 variant records; per-record QUAL, DP, FS, MQ designed to exercise each filter in the variant-filter pipeline. | Section 11.3; design exercise on the filter pipeline; project phase 3. |
| `counts_matrix.csv` | Synthetic counts matrix, 100 features by 20 cells, with three latent cell types of unequal size. | Section 11.4; simulation exercise on dimension reduction. |
| `pair_example.fasta` | Pair of short sequences (50 nt each) for the Needleman-Wunsch worked example. | Section 11.1; calculation exercise on the DP recurrence. |

## Provenance

All files are educational data generated by the editor (not from deposited
records). They are constructed to be small enough to read by eye, large
enough to exercise the algorithms, and structured so that the expected
result is verifiable from the file itself. No human-subjects data, no
clinical data, no PHI.
